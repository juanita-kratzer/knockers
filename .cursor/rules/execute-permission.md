# Execute First — User Permission

The user has given **standing permission**: anything the assistant can do or needs to do, **just do it**. No need to ask for permission before:

- Running commands (build, deploy, config, tests)
- Making code or config changes
- Creating or updating files
- Applying fixes or small improvements

When a task is clear and within scope, **execute** (run it, edit it, implement it). Only ask when something is ambiguous, requires a user choice (e.g. product decision), or needs credentials/access only the user has.
