# Australian suburb boundaries and centroids

This app can show suburb-level coverage across Australia. Suburb names and boundaries align with the **Australian Statistical Geography Standard (ASGS)** where possible.

## Data source: ABS Digital Boundary Files

The [Australian Bureau of Statistics (ABS)](https://www.abs.gov.au/statistics/standards/australian-statistical-geography-standard-asgs-edition-3/jul2021-jun2026/access-and-downloads/digital-boundary-files) publishes official digital boundaries under **ASGS Edition 3 (July 2021 – June 2026)**.

### Suburbs and Localities (SAL)

- **Product:** Suburbs and Localities – 2021  
- **Format:** GeoPackage or ESRI Shapefile (GDA2020 recommended)  
- **Download:** [ABS Digital boundary files](https://www.abs.gov.au/statistics/standards/australian-statistical-geography-standard-asgs-edition-3/jul2021-jun2026/access-and-downloads/digital-boundary-files) → **Non ABS Structures** → **Suburbs and Localities - 2021 - Shapefile** (or GeoPackage).

**Attribute fields (Shapefile):**

| Field         | Description   |
|---------------|---------------|
| `SAL_CODE21`  | 5-character code |
| `SAL_NAME21`  | Suburb/locality name |
| `STE_CODE21`  | State/territory code (1–9) |
| `STE_NAME21`  | State/territory name |

## Generating the full suburbs list for the app

The app loads **all** Australian suburbs from `public/australianSuburbs.json` when that file exists (generated from the SAL shapefile). To generate it:

1. Download the **Suburbs and Localities (SAL) 2021** Shapefile from the ABS link above (GDA2020), and unzip.
2. From the project root run: `npm run build:suburbs`  
   By default the script looks for: `$HOME/Downloads/SAL_2021_AUST_GDA2020_SHP/SAL_2021_AUST_GDA2020.shp`  
   Or pass the path: `node scripts/buildSuburbsFromShapefile.cjs /path/to/SAL_2021_AUST_GDA2020.shp`  
   Or set env: `SAL_SHP=/path/to/SAL_2021_AUST_GDA2020.shp npm run build:suburbs`
3. The script writes:
   - `public/australianSuburbs.json` (centroids + `salCode`, `state`) – the app fetches it at runtime; if missing, it falls back to a small built-in list.
   - `public/suburbBoundaries.json` (polygon rings per `salCode`) – when present, the map draws the **glow along each suburb’s real perimeter** instead of a rectangle.

## Converting to GeoJSON and centroids

1. **Download** the SAL 2021 Shapefile (zip) and unzip.
2. **Convert to GeoJSON** (e.g. [mapshaper](https://mapshaper.org/) or GDAL):
   ```bash
   ogr2ogr -f GeoJSON SAL_2021_AUST_GDA2020.json SAL_2021_AUST_GDA2020.shp
   ```
3. **Optional – centroids only:** Use a small script or mapshaper to compute the centroid of each polygon and output `{ "name": "SAL_NAME21", "lat": y, "lng": x }` so the app can use `loadSuburbDataFromGeoJSON()` without drawing full polygons.

## Using custom data in the app

- **Centroids (recommended for performance):** Place a JSON array of `{ name, lat, lng }` (and optional `count`) and import it in `Home.jsx` or via `src/data/australianSuburbs.js`. The map only needs centroids for circle markers and simple “glow” outlines.
- **Perimeter glow:** The app loads `public/suburbBoundaries.json` when available (generated by the same script). It then draws the glow along each suburb’s actual boundary polygon when zoomed in to individual suburbs. If the file is missing, no perimeter outline is drawn (only the circle markers remain).
